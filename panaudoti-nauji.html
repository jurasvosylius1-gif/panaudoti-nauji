Published URL (after enabling Pages): https://jurasvosylius1-gif.github.io/panaudoti-nauji/ How to publish (web UI) 1. Open: https://github.com/jurasvosylius1-gif/panaudoti-nauji 2. Click "Add file" → "Create new file" 3. For filename enter: index.html 4. Paste the index.html content (above) 5. In the commit box below: - Put a commit message like: Add index.html - If you see "Commit directly to the main branch" choose it; otherwise choose the default/new-branch option and then click the green button (it may say "Propose new file") 6. If you created a PR, open it and click "Merge pull request" → "Confirm merge". That creates the main branch and adds the file. 7. Repeat to add README.md (filename README.md) and paste the README content. 8. Go to Settings → Code and automation → Pages - Under "Build and deployment" choose Branch: main and folder: / (root) and click Save. 9. Wait ~1 minute and open: https://jurasvosylius1-gif.github.io/panaudoti-nauji/ How to publish (Git on your computer) 1. Save index.html and README.md into a folder. 2. Run: git init git branch -M main git remote add origin https://github.com/jurasvosylius1-gif/panaudoti-nauji.git git add index.html README.md git commit -m "Add website" git push -u origin main 3. Then enable Pages in the repository Settings → Pages (Branch: main, root). Notes - If the web editor button says "Propose new file", follow the PR flow and merge the PR; that will create the main branch. - The form sends email via FormSubmit (formsubmit.co) to the email in the form.
